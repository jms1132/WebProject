풀 푸시는 교육원에 와서 재현이나 형규와 상의 후 커밋 풀 푸시를 한다.
그전엔 절대 풀 푸시를 하지 말자.

커밋은 하면 좋지만 확실하지 않은 것은 커밋 하지 않는다.
예 ) 데이터 베이스에 회원 정보를 넣는 함수를 만드는 도중에는 커밋 하지 않는다. (풀 푸시 도 마찬가지)
    데이터 베이스에 회원 정보를 넣는 함수를 완성 했다. (완성이 되면 커밋을 해 놓는다. 풀 푸시는 상의 후 하기)

코딩을 하고 있는 도중에 풀 하면 커밋하지 않은 부분이 다 날아간다.
풀은 커밋한 내용들을 가져올 때 사용하는 것, 절대 커밋 하지 않고 풀 하지 않는다.

커밋은 함수 하나 만들었을때, 뭔가를 완성 했을때 커밋을 한다.

커밋 풀 푸시를 한 후에 항상 자신의 코드를 확인한다. 물론 깃도 확인 (충돌이 났을 수 있기 때문에)
